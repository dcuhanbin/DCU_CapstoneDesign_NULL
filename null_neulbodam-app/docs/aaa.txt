3. 상세 요구사항 (Specific Requirements)

3.1 기능 요구사항 (Functional Requirements)

3.1.1 아침/점심/저녁 출석 확인 이벤트 처리

이벤트 시간대 설정
시스템은 아침(06:00~10:00), 점심(11:00~14:00), 저녁(17:00~21:00) 3개 시간대별 출석 이벤트를 제공해야 한다. 사용자가 잠금화면 선물상자를 터치하면 현재 시간대가 이벤트 시간대에 해당하는지 확인 후 출석 처리를 수행한다. 이벤트 시간대가 아닌 경우 "아직 이벤트 시간이 아닙니다" 메시지를 표시한다.

출석 체크 처리
사용자가 지정된 시간대에 잠금화면 선물상자를 터치하면 해당 시간대 출석이 완료되어야 한다. 시스템은 현재 시간대를 확인하고, 해당 시간대 출석 여부를 중복 체크한 후, 출석 완료 시 DB에 사용자 ID, 출석 시간대, 타임스탬프를 기록한다. 출석 완료 애니메이션 및 획득 캐시를 표시하며, 이미 출석한 시간대인 경우 "이미 출석하셨습니다" 메시지를 표시한다.

출석 보상 지급
출석 완료 시 시간대별로 정해진 캐시를 자동으로 지급해야 한다. 기본 출석 캐시(예: 시간대당 50캐시)를 지급하고, 연속 출석 보너스를 계산하여 추가 지급한다. 사용자 총 캐시 잔액을 업데이트하고, 지급된 캐시 금액 및 현재 잔액을 표시한다. 서버 오류 시 재시도 로직을 수행하며, 3회 실패 시 로컬에 임시 저장한다.

연속 출석 보너스
하루 3회 출석을 모두 완료한 날이 연속될 경우 추가 보너스를 제공해야 한다. 시스템은 일일 출석 완료 데이터를 기반으로 연속 출석 일수를 계산하고, 연속 일수에 따른 보너스 캐시를 산정한다(예: 7일 연속 시 500캐시). 연속 출석 일수 및 보너스 캐시를 안내하며, 하루라도 3회 미만 출석 시 연속 일수를 초기화한다.

미출석 알림
각 시간대 종료 30분 전 미출석 사용자에게 알림을 전송해야 한다. 스케줄러가 자동으로 해당 시간대 미출석 사용자 목록을 조회하고, FCM/APNs를 통해 "아침/점심/저녁 출석 체크를 잊지 마세요!" 푸시 알림을 전송한다. 알림 권한이 없는 경우 알림 전송을 생략한다.



3.1.2 무응답 누적 감지 및 보호자 알림 발송

무응답 상태 판단 기준
시스템은 사용자의 무응답 상태를 명확한 기준에 따라 판단해야 한다. 연속 2회 이상 출석 미완료, 또는 최근 12시간 동안 잠금화면 반응 없음, 또는 최근 24시간 동안 앱 실행 기록 없음 중 하나라도 충족되면 무응답 상태로 플래그를 설정한다. 사용자별 마지막 활동 시간을 모니터링하고, 무응답 감지 로그를 DB에 저장한다. 사용자가 수동으로 "일시정지 모드"를 설정한 경우 무응답 감지를 비활성화한다.

보호자 알림 발송 조건
무응답 상태가 감지되면 등록된 보호자에게 즉시 알림을 전송해야 한다. 등록된 보호자 연락처를 조회하고, 우선순위에 따라 1차 보호자에게 문자를 발송한다. 1차 보호자 무응답 시 30분 후 2차 보호자에게 발송한다. 문자 내용은 "OOO님이 12시간 이상 앱 활동이 없습니다. 안부를 확인해주세요."와 같은 형식으로 전달된다. 보호자가 미등록된 경우 사용자에게 보호자 등록 안내 알림을 전송하며, 발송 실패 시 최대 3회까지 재전송을 시도한다.

보호자 알림 내용 구성
보호자에게 전송되는 알림은 명확하고 행동 가능한 정보를 포함해야 한다. 사용자 이름, 마지막 활동 시간, 무응답 감지 사유(출석 미완료, 잠금화면 미반응 등), 사용자 연락 권장 문구를 포함한다. 예시: "김철수님이 24시간 동안 앱 활동이 없습니다. 마지막 활동: 12월 1일 오전 8시. 안부를 확인해주세요." 문자 발송 실패 시 앱 내 알림으로 대체 전송한다.

무응답 해제 처리
무응답 상태의 사용자가 다시 활동을 시작하면 자동으로 무응답 상태를 해제해야 한다. 사용자의 출석, 잠금화면 해제, 앱 실행 이벤트가 발생하면 무응답 플래그를 해제하고, 보호자에게 "김철수님이 앱 활동을 재개했습니다"와 같은 안전 확인 알림을 전송한다. 무응답 해제 시간을 기록하며, 해제 알림 전송 실패 시 로그만 기록한다.

무응답 통계 및 기록
사용자의 무응답 발생 이력을 통계로 제공해야 한다. 누적된 무응답 기록을 바탕으로 월별/주별 무응답 발생 횟수를 집계하고, 평균 무응답 지속 시간을 계산한다. 통계 화면에서 무응답 패턴을 시각화하며, 데이터가 부족한 경우 "아직 충분한 데이터가 없습니다" 메시지를 표시한다.



3.1.3 잠금화면 슬라이드 및 광고 시청 기반 캐시 적립

잠금화면 캐시 적립
사용자가 잠금화면을 슬라이드 해제할 때마다 캐시를 적립해야 한다. 슬라이드 동작을 감지하고, 일일 최대 적립 횟수를 확인한다(예: 하루 50회 제한). 적립 조건 충족 시 슬라이드당 정해진 캐시를 지급하고(예: 2캐시), 사용자 잔액을 업데이트한다. 잠금화면 해제 시 "+2 캐시" 애니메이션을 표시하며, 일일 한도 초과 시 "오늘의 잠금화면 캐시 적립이 완료되었습니다" 메시지를 표시한다.

광고 시청 캐시 적립
사용자가 제휴 광고를 시청 완료하면 추가 캐시를 지급해야 한다. 광고 시청 완료 여부를 SDK 콜백으로 검증하고, 일일 광고 시청 횟수를 확인한다(예: 하루 10회 제한). 광고 종류에 따라 차등 캐시를 지급한다(예: 15초 광고 = 10캐시, 30초 광고 = 20캐시). 광고 종료 후 "+10 캐시 적립!" 팝업을 표시하며, 광고 중도 이탈 시 캐시를 미지급한다. 네트워크 오류 시 "광고를 불러올 수 없습니다" 안내를 표시한다.

캐시 적립 일일 한도 관리
시스템은 잠금화면 및 광고 시청을 통한 일일 캐시 적립 한도를 관리해야 한다. 캐시 적립 요청 시 금일 누적 적립량을 조회하고, 적립 유형별 한도를 확인한다(잠금화면: 100캐시/일, 광고: 200캐시/일). 한도 내 적립 가능 여부를 판단하며, 한도 초과 시 "오늘의 적립 한도에 도달했습니다. 내일 다시 만나요!" 메시지를 표시한다. 자정(00:00) 기준으로 일일 한도를 자동 초기화한다.

잠금화면 광고 배너 표시
잠금화면에 간단한 광고 배너를 표시하여 추가 수익원을 확보해야 한다. 잠금화면 활성화 시 광고 서버에서 배너 이미지 및 링크를 수신하고, 잠금화면 하단 또는 상단에 배너를 렌더링한다. 사용자 터치 시 선택적으로 광고 페이지로 이동할 수 있다. 광고 로드 실패 시 기본 이미지를 표시한다.

캐시 적립 내역 기록
모든 캐시 적립 내역을 시간순으로 기록하고 사용자가 확인할 수 있어야 한다. 적립 시간, 적립 사유, 적립 금액을 DB에 저장하고, 사용자별 적립 내역 리스트를 관리한다. "캐시 내역" 화면에서 날짜별 적립 내역을 표시하며, 내역이 없는 경우 "아직 적립 내역이 없습니다" 안내를 제공한다.



3.1.4 상점 포인트 교환 및 쿠폰 관리 기능

상점 상품 목록 제공
사용자가 캐시로 교환 가능한 상품 및 쿠폰 목록을 제공해야 한다. 상점 화면 진입 시 서버에서 상품 리스트를 조회하고, 카테고리별로 분류한다(편의점 쿠폰, 카페 쿠폰, 기프티콘 등). 각 상품의 필요 캐시와 재고 상태를 표시하며, 상품 이미지, 이름, "구매하기" 버튼을 제공한다. 상품 품절 시 "일시 품절" 표시를 하고, 네트워크 오류 시 캐시된 이전 목록을 표시한다.

상품 구매 처리
사용자가 충분한 캐시를 보유한 경우 상품을 구매할 수 있어야 한다. 상품 선택 및 "구매하기" 버튼 클릭 시 사용자 캐시 잔액과 상품 재고를 확인한다. 조건 충족 시 캐시를 차감하고 쿠폰을 발급하며, 거래 내역을 DB에 저장한다. "구매가 완료되었습니다. 내 쿠폰함에서 확인하세요" 안내를 표시한다. 캐시 부족 시 "캐시가 부족합니다. 현재 잔액: OOO캐시" 메시지를, 재고 부족 시 "상품이 일시 품절되었습니다" 안내를 표시한다.

쿠폰함 관리
구매한 쿠폰을 "내 쿠폰함"에서 확인하고 사용할 수 있어야 한다. 쿠폰함 화면 진입 시 사용자가 보유한 쿠폰 리스트를 조회하고, 쿠폰 상태별로 분류한다(사용 가능, 사용 완료, 유효기간 만료). 쿠폰 상세 정보로 바코드, QR코드, 사용처, 유효기간을 제공한다. 보유 쿠폰이 없는 경우 "쿠폰함이 비어있습니다" 메시지를 표시한다.

쿠폰 사용 처리
사용자가 쿠폰을 사용하면 상태를 업데이트해야 한다. 쿠폰 "사용하기" 버튼 클릭 시 바코드/QR코드 화면을 표시하고, 사용자 확인 후 "사용 완료" 처리를 수행한다. 쿠폰 상태를 '사용 완료'로 업데이트하고 사용 일시를 기록한다. 바코드/QR코드 화면 및 "사용 완료되었습니다" 확인 메시지를 표시하며, 이미 사용된 쿠폰의 재사용을 방지하고 유효기간 만료 쿠폰 사용을 불가 처리한다.

쿠폰 유효기간 관리
쿠폰의 유효기간을 관리하고 만료 예정 쿠폰을 사용자에게 알려야 한다. 스케줄러가 일일 체크를 수행하여 유효기간 3일 이내 쿠폰을 조회하고, 해당 사용자에게 푸시 알림을 전송한다. 유효기간 만료된 쿠폰은 자동으로 상태를 변경한다. "OO 쿠폰이 3일 후 만료됩니다" 알림을 전송하며, 알림 권한이 없는 경우 앱 내 알림으로 대체한다.



3.1.5 알람 내역·캐시 내역·보행 통계 제공

알람 내역 제공
사용자가 수신한 모든 알림 내역을 날짜별로 확인할 수 있어야 한다. 알람 내역 화면 진입 시 사용자별 알림 기록을 조회하고, 알림 유형별로 분류한다(출석 알림, 보호자 알림, 시스템 공지). 날짜 역순으로 정렬하여 알림 제목, 내용, 발송 시간 리스트를 제공한다. 알림 내역이 없는 경우 "알림 내역이 없습니다" 메시지를 표시한다.

캐시 내역 제공
사용자의 캐시 적립 및 사용 내역을 상세히 제공해야 한다. 캐시 내역 화면 진입 시 적립/사용 내역을 조회하고, 거래 유형별로 분류한다(적립, 사용, 환불). 금액, 잔액, 발생 시간을 표시하며, 예시로 "+50 캐시 | 아침 출석 체크 | 12/01 08:30", "-500 캐시 | 스타벅스 아메리카노 구매 | 12/01 14:20"와 같은 형식으로 제공한다. 내역이 없는 경우 "아직 거래 내역이 없습니다" 안내를 표시한다.

보행 통계 제공
사용자의 일일/주간 보행 데이터를 시각화하여 제공해야 한다. 통계 화면 진입 시 걸음 수, 이동 거리(km), 소모 칼로리, 보행 시간을 집계하고, 일일/주간 데이터 그래프를 생성한다. 목표 대비 달성률을 계산하여 표시한다. "오늘 걸음 수: 8,523걸음", "이동 거리: 6.4km", "소모 칼로리: 320kcal"와 같은 정보를 제공하고, 주간 그래프를 막대 또는 선 그래프로 표시한다. 걸음 수 데이터가 없는 경우 "활동 데이터를 수집 중입니다" 안내를, 권한 미허용 시 "만보기 권한을 허용해주세요" 안내를 표시한다.

주간 통계 그래프
최근 7일간의 보행 데이터를 그래프로 시각화해야 한다. 통계 화면 내 "주간" 탭 선택 시 최근 7일 일별 걸음 수를 조회하고, 막대 그래프 또는 선 그래프로 렌더링한다. 평균 걸음 수를 계산하여 요일별 걸음 수 막대 그래프 및 "주간 평균: 7,234걸음"을 표시한다. 7일 미만 데이터인 경우 수집된 데이터만 표시한다.

통계 데이터 갱신
보행 통계는 실시간으로 업데이트되어야 한다. 백그라운드 걸음 수 센서 데이터를 주기적으로 수집하고(예: 5분마다), 일일 누적 걸음 수를 계산한다. 이동 거리 및 칼로리를 환산하여 DB를 업데이트하며, 통계 화면 진입 시 최신 데이터를 표시한다. 센서 오류 시 "걸음 수를 측정할 수 없습니다" 안내를 표시한다.



3.1.6 계정 관리 및 보호자 정보 등록 기능

사용자 계정 생성
사용자는 이메일 또는 전화번호로 계정을 생성할 수 있어야 한다. 이메일/전화번호, 비밀번호, 이름, 생년월일을 입력받아 유효성을 검증한다(이메일 형식, 비밀번호 강도 등). 중복 계정을 확인하고, 계정 생성 후 DB에 저장한다. 인증 메일/문자를 발송하고, "회원가입이 완료되었습니다" 안내 및 로그인 화면으로 이동한다. 중복 계정 시 "이미 가입된 이메일입니다" 메시지를, 인증 실패 시 재전송 옵션을 제공한다.

사용자 프로필 관리
사용자는 프로필 정보를 수정할 수 있어야 한다. 이름, 프로필 사진, 전화번호 등을 입력받아 기존 정보를 불러오고, 수정 사항의 유효성을 검증한다. DB를 업데이트하고 "프로필이 수정되었습니다" 확인 메시지를 표시한다. 필수 항목 누락 시 "필수 정보를 입력해주세요" 안내를 표시한다.

보호자 등록
사용자는 비상 연락을 받을 보호자를 등록할 수 있어야 한다. 보호자 이름, 전화번호, 관계, 우선순위를 입력받아 유효성을 검증한다(전화번호 형식). 최대 3명까지 보호자를 등록할 수 있으며, 우선순위를 설정한다(1차, 2차, 3차). DB에 저장 후 "보호자가 등록되었습니다" 확인 메시지를 표시한다. 중복 전화번호 입력 시 "이미 등록된 번호입니다" 안내를, 3명 초과 등록 시 "최대 3명까지 등록 가능합니다" 메시지를 표시한다.

보호자 정보 수정 및 삭제
등록된 보호자 정보를 수정하거나 삭제할 수 있어야 한다. 보호자 목록을 조회하고, 선택한 보호자 정보를 수정 또는 삭제한다. 삭제 시 우선순위를 재조정하고 DB를 업데이트한다. "보호자 정보가 수정/삭제되었습니다" 확인 메시지를 표시하며, 삭제 시 "정말 삭제하시겠습니까?" 확인 팝업을 표시한다.

비상 알람 설정
사용자는 비상 알람 발송 조건을 직접 설정할 수 있어야 한다. 무응답 기준 시간, 연속 미출석 횟수 등을 입력받아 기본 설정 값을 제공한다(예: 12시간 무응답, 2회 연속 미출석). 사용자 맞춤 설정을 저장하고, 설정 변경 시 선택적으로 보호자에게 안내 문자를 발송한다. "비상 알람 설정이 저장되었습니다" 확인 메시지를 표시하며, 잘못된 값 입력 시 "올바른 값을 입력해주세요" 안내를 표시한다.

로그인 및 로그아웃
사용자는 안전하게 로그인하고 로그아웃할 수 있어야 한다. 이메일/전화번호와 비밀번호를 입력받아 DB와 대조하고, 인증 성공 시 세션 토큰을 발급한다. 자동 로그인 설정을 선택적으로 저장할 수 있다. 로그인 성공 시 홈 화면으로 이동하며, 로그인 실패 시 "이메일 또는 비밀번호가 일치하지 않습니다" 메시지를 표시한다. 5회 연속 실패 시 계정을 일시 잠금한다.

3.2 사용자 인터페이스 요구사항 (UI/UX Requirements)

3.2.1 출석 체크 UI (아침·점심·저녁 이벤트)

잠금화면 선물상자 디자인
잠금화면에 시간대별로 다른 디자인의 선물상자 아이콘을 표시해야 한다. 아침(06:00~10:00)에는 해 뜨는 이미지와 노란색 선물상자를, 점심(11:00~14:00)에는 햇빛 이미지와 주황색 선물상자를, 저녁(17:00~21:00)에는 달 이미지와 보라색 선물상자를 표시한다. 선물상자 터치 시 확대 후 터지는 애니메이션 효과를 제공하고, 출석 완료 시 체크 표시 및 획득 캐시를 표시한다. 아이콘 크기는 최소 60x60dp로 설정하고, 고대비 색상을 사용하며, 터치 영역을 패딩을 포함하여 확장한다.

출석 완료 피드백
출석 완료 시 명확한 시각적·청각적 피드백을 제공해야 한다. 선물상자 터짐 애니메이션(0.5초), "+50 캐시" 텍스트 상승 효과, 체크 마크 표시를 시각적 요소로 제공한다. 청각적 요소로 긍정적인 효과음을 선택적으로 재생하며(음소거 가능), "아침 출석 완료! 50캐시를 받았어요"와 같은 메시지를 표시한다. 진동 피드백(햅틱)을 제공하여 접근성을 향상시킨다.

출석 현황 표시
홈 화면에서 오늘의 출석 현황을 한눈에 확인할 수 있어야 한다. 아침/점심/저녁 출석 상태(완료/미완료)를 각 시간대별 체크 아이콘과 함께 표시하고, 연속 출석 일수를 표시한다. 가로 정렬로 3개 시간대를 표시하며, 완료 시 컬러 아이콘을, 미완료 시 그레이스케일을 적용한다. 미완료 시간대 터치 시 알림 설정 화면으로 이동한다.



3.2.2 잠금화면 슬라이드 캐시 인터페이스

잠금화면 레이아웃
잠금화면은 시간, 날짜, 캐시 정보를 명확히 표시해야 한다. 상단에 시간(HH:MM)과 날짜(MM월 DD일 요일)를, 중앙에 선물상자 아이콘(출석 이벤트용)을, 하단에 "밀어서 잠금해제 → +2 캐시" 안내 텍스트를 배치한다. 하단 배너 영역에 선택적으로 광고를 표시한다. 시간은 48sp, 날짜는 18sp, 안내 텍스트는 16sp 크기로 설정하고, 흰색 텍스트와 반투명 검은색 배경 오버레이를 적용한다.

슬라이드 인터랙션
슬라이드 동작이 직관적이고 명확해야 한다. 하단에서 상단으로 또는 좌에서 우로 슬라이드 방향을 설정한다. 슬라이드 시작 시 캐시 아이콘이 따라다니고, 슬라이드 완료 시 "+2 캐시" 팝업 애니메이션을 표시한다. 실패 시 원위치로 복귀한다. 진동 및 효과음 피드백을 제공하며, 설정에서 끄기 가능하도록 한다.

광고 배너 표시
잠금화면 하단에 간단한 광고 배너를 표시해야 한다. 화면 너비 × 60dp 높이로 배너 크기를 설정하고, 화면 하단에 고정한다. 터치 시 선택적으로 광고 상세 페이지로 이동할 수 있다. 슬라이드 동작을 방해하지 않도록 제약을 설정한다.



3.2.3 상점 및 쿠폰함 UI

상점 메인 화면
상점 화면은 상품을 카테고리별로 명확히 분류하여 표시해야 한다. 상단에 내 캐시 잔액을 큰 글씨로 표시하고, 카테고리 탭(편의점, 카페, 외식, 기프티콘)을 제공한다. 상품 카드에는 이미지, 이름, 필요 캐시, "구매" 버튼을 포함한다. 그리드 형식(1행당 2개 상품)으로 레이아웃을 구성하고, 인기순, 낮은 가격순, 높은 가격순 필터를 제공한다.

상품 상세 화면
상품 선택 시 상세 정보를 제공해야 한다. 상품 이미지(확대 가능), 상품명, 브랜드, 필요 캐시, 사용처, 유효기간, 주의사항을 표시한다. "구매하기" 버튼은 충분한 캐시 보유 시에만 활성화한다. "OOO 캐시를 사용하여 구매하시겠습니까?"와 같은 확인 팝업을 표시한다.

쿠폰함 화면
보유한 쿠폰을 상태별로 확인할 수 있어야 한다. 사용 가능, 사용 완료, 유효기간 만료 탭으로 구성한다. 쿠폰 카드에는 브랜드 로고, 쿠폰명, 유효기간, "사용하기" 버튼(사용 가능한 쿠폰만)을 포함한다. 유효기간 임박순으로 정렬한다.

쿠폰 사용 화면
쿠폰 사용 시 바코드/QR코드를 명확히 표시해야 한다. 쿠폰명 및 브랜드, 바코드/QR코드(화면 중앙, 최대 크기), 쿠폰 번호(텍스트), 유효기간, "사용 완료" 버튼을 표시한다. 바코드 인식을 위해 화면 밝기를 자동으로 최대로 설정하고, 사용 화면 진입 시 화면 자동 꺼짐을 비활성화한다.



3.2.4 통계 화면 (주간 그래프·보행 데이터 시각화)

통계 메인 화면
오늘의 활동 데이터를 요약하여 표시해야 한다. 오늘 걸음 수를 큰 글씨로 중앙에 배치하고, 이동 거리(km), 소모 칼로리(kcal), 보행 시간(분)을 표시한다. 원형 진행 바로 목표 대비 달성률을 시각화하고, 각 지표를 아이콘과 함께 표시한다. 긍정적인 느낌의 파란색/초록색 계열 색상을 사용한다.

주간 그래프
최근 7일간의 활동 추이를 그래프로 표시해야 한다. 막대 그래프 또는 선 그래프 유형을 사용하고, X축에 요일(월~일)을, Y축에 걸음 수를 표시한다. 주간 평균, 최고/최저 기록을 추가 정보로 표시한다. 각 막대 터치 시 해당 일의 상세 데이터 팝업을 표시한다.

월간 통계
월별 활동 추이를 확인할 수 있어야 한다. 히트맵 또는 캘린더 형식으로 표시하고, 활동량에 따라 연한색에서 진한색으로 그라데이션 색상을 적용한다. 특정 날짜 터치 시 상세 정보를 표시한다.



3.2.5 알람·캐시 내역 화면

알람 내역 화면
수신한 알림을 시간순으로 확인할 수 있어야 한다. 리스트에 알림 아이콘(유형별 다른 아이콘), 알림 제목(볼드체), 알림 내용(일반체, 2줄까지 표시), 수신 시간(상대 시간 표시: "3시간 전")을 포함한다. 최신순으로 정렬하고, 전체, 출석 알림, 보호자 알림, 시스템 공지 필터를 제공한다. 알림 터치 시 상세 내용을 확장한다.

캐시 내역 화면
캐시 적립 및 사용 내역을 명확히 구분하여 표시해야 한다. 리스트에 거래 유형(적립: +, 사용: -), 금액(적립: 파란색, 사용: 빨간색), 사유(예: "아침 출석", "스타벅스 쿠폰 구매"), 거래 후 잔액, 거래 일시를 포함한다. 최신순으로 정렬하고, 전체, 적립만, 사용만 필터를 제공한다. 오늘, 최근 7일, 최근 30일, 전체 기간 선택 옵션을 제공한다.

빈 상태 처리
내역이 없는 경우 안내 메시지를 표시해야 한다. 일러스트 이미지(빈 박스 등), "아직 알림/캐시 내역이 없습니다" 메시지, 관련 기능 안내 버튼(예: "출석하러 가기")을 포함한다. 중앙 정렬하고 부드러운 색상을 사용한다.



3.2.6 계정 및 보호자 등록 화면

프로필 화면
사용자 정보를 확인하고 수정할 수 있어야 한다. 프로필 사진(원형, 터치 시 수정 가능), 이름, 이메일/전화번호, 가입일을 표시한다. 메뉴 리스트에 프로필 수정, 보호자 관리, 비상 알람 설정, 로그아웃을 포함한다. 리스트 형식으로 디자인하고, 각 항목에 우측 화살표 아이콘을 배치한다.

보호자 등록 화면
보호자 정보를 간편하게 입력할 수 있어야 한다. 보호자 이름(텍스트 입력), 전화번호(숫자 키패드), 관계(드롭다운: 자녀, 형제자매, 친구, 기타), 우선순위(자동 할당, 순서 조정 가능) 입력 필드를 제공한다. "등록하기", "취소" 버튼을 배치하고, 전화번호 형식과 중복을 검증한다. 등록 완료 시 토스트 메시지를 표시한다.

보호자 목록 화면
등록된 보호자를 확인하고 관리할 수 있어야 한다. 리스트에 우선순위 번호(1차, 2차, 3차), 보호자 이름, 전화번호(일부 마스킹: 010-1234-5678), 관계, "수정/삭제" 버튼을 포함한다. 드래그로 우선순위를 변경하고, 스와이프로 삭제할 수 있다. 최대 3명까지 등록 가능하며, 초과 시 "추가" 버튼을 비활성화한다.

비상 알람 설정 화면
비상 알람 발송 조건을 설정할 수 있어야 한다. 무응답 기준 시간(슬라이더: 6~24시간), 연속 미출석 기준(스테퍼: 1~5회), 알림 발송 시간대 제한(스위치: ON/OFF) 설정 항목을 제공한다. "12시간 동안 활동이 없으면 보호자에게 알림이 전송됩니다"와 같은 안내 문구를 표시한다. 화면 하단에 저장 버튼을 고정하고, "설정이 저장되었습니다" 확인 팝업을 표시한다.

로그인 화면
간편하고 안전한 로그인 UI를 제공해야 한다. 이메일/전화번호와 비밀번호 입력 필드를 제공하고, 비밀번호는 마스킹 처리하며 표시/숨김 토글을 제공한다. 자동 로그인 체크박스와 비밀번호 찾기 링크를 포함한다. "로그인", "회원가입" 버튼을 배치하고, 선택적으로 카카오, 네이버, 구글 소셜 로그인을 제공한다. 큰 터치 영역과 명확한 레이블을 사용하여 접근성을 향상시킨다.

 3.3 구현 우선순위 (Development Priority)

3.3.1 출석 체크·무응답 감지·보호자 알림

본 서비스의 핵심 가치인 안전 확인 기능으로, 가장 높은 우선순위로 개발되어야 한다. 사용자의 생존 및 안전 여부를 확인하는 것이 프로젝트의 근본 목적이므로, 아침/점심/저녁 출석 이벤트 처리, 무응답 상태 감지, 보호자 알림 발송 기능을 최우선적으로 구현해야 한다.

잠금화면에 시간대별 선물상자를 표시하고, 사용자가 터치하면 출석이 완료되는 기본 흐름을 구현한다. 출석 데이터를 DB에 저장하고, 설정된 기준(연속 2회 미출석, 12시간 무응답 등)에 따라 무응답 상태를 판단하는 로직을 개발한다. 무응답 감지 시 등록된 보호자에게 문자 메시지를 자동 발송하는 알림 시스템을 구축한다.

이 단계에서 구현되어야 할 주요 기능은 출석 이벤트 시간대 설정 및 처리, 출석 완료 피드백 UI, 무응답 판단 알고리즘, 보호자 연락처 관리, 문자 발송 API 연동이다. MVP(Minimum Viable Product) 단계에서 반드시 작동해야 하는 핵심 기능이므로, 안정성과 정확성을 최우선으로 개발한다.



3.3.2 캐시 적립 기능·계정 관리

사용자의 지속적인 참여를 유도하는 리워드 시스템과 기본적인 사용자 관리 기능을 구현하는 단계이다. 핵심 안전 기능이 완성된 후, 사용자가 서비스를 자발적으로 사용하도록 동기를 부여하는 캐시 적립 체계를 개발한다.

잠금화면 슬라이드 시 캐시가 적립되는 기능을 구현하고, 광고 시청 완료 시 추가 캐시를 지급하는 시스템을 구축한다. 일일 적립 한도를 관리하고, 캐시 적립 내역을 DB에 기록하는 로직을 개발한다. 사용자 계정 생성, 로그인/로그아웃, 프로필 관리 기능을 구현하여 개인화된 서비스를 제공할 수 있는 기반을 마련한다.

보호자 등록 화면을 개발하여 사용자가 최대 3명의 보호자 정보를 입력하고 우선순위를 설정할 수 있도록 한다. 비상 알람 설정 화면을 통해 무응답 기준 시간과 연속 미출석 기준을 사용자가 직접 조정할 수 있는 기능을 제공한다.

이 단계에서는 잠금화면 슬라이드 인터페이스, 광고 SDK 연동, 캐시 적립 로직, 회원가입 및 로그인 UI/UX, 보호자 등록 및 관리 화면, 비상 알람 설정 기능이 구현되어야 한다. 사용자 온보딩 경험을 완성하고, 기본적인 리워드 순환 구조를 확립하는 것이 목표이다.



3.3.3 통계·내역·로그 수집

사용자 경험을 향상시키고 데이터 기반으로 서비스를 개선하기 위한 정보 제공 기능을 개발하는 단계이다. 사용자가 자신의 활동 패턴을 확인하고, 적립 및 사용 내역을 투명하게 조회할 수 있도록 한다.

알람 내역 화면을 개발하여 사용자가 수신한 모든 알림을 날짜별로 확인할 수 있도록 하고, 캐시 내역 화면을 통해 적립 및 사용 거래를 상세히 조회할 수 있게 한다. 보행 통계 기능을 구현하여 걸음 수, 이동 거리, 소모 칼로리, 보행 시간을 시각화한다. 주간 그래프를 통해 최근 7일간의 활동 추이를 막대 또는 선 그래프로 표시한다.

백그라운드에서 걸음 수 센서 데이터를 주기적으로 수집하고, 이를 DB에 저장하여 실시간으로 통계를 업데이트하는 시스템을 구축한다. 사용자 행동 패턴을 분석할 수 있는 로그 수집 체계를 마련하여, 향후 서비스 개선과 맞춤형 기능 개발에 활용할 수 있는 데이터를 확보한다.

이 단계에서는 알람 내역 조회 및 필터링, 캐시 내역 조회 및 기간 선택, 보행 데이터 수집 및 저장, 통계 그래프 시각화, 주간/월간 통계 화면이 구현되어야 한다. 사용자에게 자신의 활동을 되돌아볼 수 있는 기회를 제공하고, 서비스 사용에 대한 만족도를 높이는 것이 목표이다.



3.3.4 상점·쿠폰 기능

적립한 캐시를 실제 가치로 전환할 수 있는 교환 시스템을 구축하여 서비스의 완성도를 높이는 단계이다. 사용자가 모은 캐시를 상품이나 쿠폰으로 교환할 수 있는 상점 기능을 개발하여, 리워드 시스템의 선순환 구조를 완성한다.

상점 메인 화면을 개발하여 카테고리별(편의점, 카페, 외식, 기프티콘)로 상품을 분류하고, 상품 이미지, 이름, 필요 캐시, 재고 상태를 표시한다. 상품 상세 화면에서 사용처, 유효기간, 주의사항 등의 정보를 제공하고, 구매 프로세스를 구현한다. 사용자 캐시 잔액을 확인하고, 충분한 캐시 보유 시 구매를 진행하며, 쿠폰을 발급하여 DB에 저장한다.

쿠폰함 화면을 개발하여 보유 쿠폰을 사용 가능, 사용 완료, 유효기간 만료 상태별로 분류하고, 바코드/QR코드를 통해 쿠폰을 사용할 수 있는 인터페이스를 제공한다. 쿠폰 유효기간을 관리하고, 만료 예정 쿠폰에 대해 사용자에게 알림을 전송하는 스케줄러를 구축한다.

제휴 업체와의 연동을 위한 API를 개발하고, 쿠폰 발급 및 사용 처리 로직을 구현한다. 상점 상품 목록 관리, 재고 관리, 결제(캐시 차감) 처리 시스템을 구축한다.

이 단계에서는 상점 UI 및 상품 목록 API, 상품 구매 및 캐시 차감 로직, 쿠폰 발급 및 DB 저장, 쿠폰함 조회 및 상태 관리, 바코드/QR코드 생성 및 표시, 쿠폰 유효기간 관리 스케줄러가 구현되어야 한다. 사용자가 적립한 캐시를 실생활에서 활용할 수 있는 실질적인 가치를 제공하고, 서비스 사용 만족도를 극대화하는 것이 목표이다.



개발 진행 원칙

각 우선순위 단계는 이전 단계의 기능이 안정적으로 구현되고 테스트를 완료한 후 진행하는 것을 원칙으로 한다. 사용자 피드백 및 베타 테스트 결과에 따라 우선순위가 조정될 수 있으며, 각 단계별로 충분한 QA(Quality Assurance)를 거쳐 안정성을 확보한 후 다음 단계로 넘어간다.

1순위 기능은 서비스 런칭 전 반드시 완성되어야 하는 필수 기능이며, 2순위 기능은 정식 출시 시 포함되어야 하는 핵심 기능이다. 3순위와 4순위 기능은 출시 후 점진적으로 업데이트하여 추가할 수 있으나, 가능한 한 초기 버전에 포함하여 완성도 높은 서비스를 제공하는 것을 목표로 한다.
